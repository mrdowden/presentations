Apache JMeter

Test performance and simulate load over various protocols:

  • HTTP, HTTPS, SOAP (Web)
  • FTP
  • JDBC (Database)
  • LDAP
  • JMS (Middleware)
  • SMTP, POP, IMAP (Email)
  • MongoDB
  • TCP
  • Shell Scripts
JMeter may be used for Performance, Load, and Stress testing. Performance test "best possible" response under specified configuration and conditions. Load test performance with concurrent requests, profile the system - how components scale with traffic. Stress identify the breaking point of the system (max load).

Apache JMeter

Features

  • Multithreaded
  • Distributed
  • Integrated recording
  • Error handling
  • Timing & Scheduling
  • Extensible
  • Scriptable
Why JMeter? Why not build something?
I see JMeter as a pluggable thread-management platform. The core feature is its ability to spawn and synchronize threads, including the recording of data they generate. JMeter makes it easy to multithread requests, collaborate across multiple machines, record results in combination across all testing nodes, and it handles errors gracefully. Also, as you are about to see, getting started with JMeter is not that hard.

Getting Started

  • Download: http://jmeter.apache.org/
  • Extract folder from ZIP or TAR.GZ
  • Run bin/jmeter.sh or bin/jmeter.bat
Download (binary) of JMeter is a ZIP or TAR.GZ containing a single folder.

Basic JDBC Test

  • JDBC Connection Configuration
  • Thread Group
  • JDBC Request Sampler
  • Summary Report Listener
Before running a JDBC test you'll need to install the JDBC Driver JAR. To do this, simply copy the JAR into the /lib/ folder of JMeter. Alternatively, you may modify the user.classpath in jmeter.properties to include the location of your JDBC Driver JAR.
I am using the free MaxMind GeoLite IP Geolocation database for testing. I am also using Derby for demonstration since it is a small, well contained database which doesn't impact your system configuration.
Due to a limitation of Derby I am keeping the thread count at 1.
Pay close attention to the "variable name" field, which here is set to "derby". This name will be used again on the JDBC Sampler to associate with these connection details.
